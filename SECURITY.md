# Security Policy

## Supported versions

Only the latest state of `main` is currently supported.

## Reporting a vulnerability

Please do not open public issues for security reports.

Report privately using one of these options:

- GitHub Security Advisories (preferred)
- Direct contact with repository maintainers

Include as much detail as possible:

- Impact and severity estimate
- Reproduction steps or proof of concept
- Affected routes/endpoints/files
- Suggested fix (if available)

## Response target

- Initial triage: up to 5 business days
- Status update after triage: up to 10 business days
